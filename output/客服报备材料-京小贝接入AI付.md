## 1. 已接入场景

**AI付（金络收银台）** 目前已接入京东金融多个业务场景，形成闭环AI支付体验。本期**京小贝场景**作为新的接入方，将京东AI付能力接入京小贝AI助手的白条还款支付链路。

**当前接入场景列表：**

| 场景 | 状态 | 说明 |
|------|------|------|

| 京小贝白条还款 | 🆕 **本次新增** | 京小贝AI助手场景下的白条还款 |

## 2. 产品介绍

### 2.1 产品定位

**京小贝**是京东金融的AI助手，统一的金融大脑中枢（Master），支撑全业务线服务体验升级。京小贝场景接入金融各业务场景，作为统一AI助手接入和外部对接。

小贝中包含：白条还款、小金库转入、基金申购等各类支付诉求。**京东AI付**定位为闭环AI场景支付体验，二者结合有利于小贝场景各业务体验闭环。

### 2.2 本期范围

**一期接入：白条账单还款**

本次聚焦 **京小贝场景→白条还款** 链路，打造AI原生闭环的还款支付体验。

### 2.3 产品形态

- **金融场景AI付产品形态：** 当前场景配置「不免密、不轮询扣款」
- **金融场景支付工具：** 银行卡、银行转账、钱包余额、小金库（开通&转入小金库）
  - 本期**不支持**分期、数字人民币、组合支付
- **场景属性：** 白条还款场景 客单1200→不免密

## 3. 开通流程

### 3.1 开通前置条件

| 条件 | 说明 |
|------|------|
| 用户需为京小贝已注册用户 | 通过京小贝Agent进入支付流程 |
| 用户有京东支付账户 | 需完成实名认证 |
| 支付SDK需接入 | 京小贝场景需要接入支付SDK核验 |

### 3.2 开通步骤

1. **用户触发：** 用户在京小贝场景下查询白条账单，进入还款意图
2. **Agent识别：** 小贝MA→消金Agent→支付Agent 识别支付意图
3. **AI付获取：** 支付Agent 调AI付接口，获取收银台信息
4. **开通检测：** 检测用户是否已开通AI付
   - 已开通：展示AI付收银台卡片
   - 未开通：展示开通协议，用户勾选同意后开通
5. **开通完成：** 返回AI付优先付款方式、支付方式、开通协议、按钮文案

## 4. 支付流程

### 4.1 主链路（两条）

| 链路 | 路径 | 说明 |
|------|------|------|
| **链路A：白条查账还款AI付** | 小贝MA→消金Agent（白条还款卡片）→支付Agent（AI付卡片） | 用户先查账→确认→进入支付 |
| **链路B：一句话白条还款** | 小贝MA→消金Agent→支付Agent（AI付卡片） | 用户直接一句话引导还款→进入支付 |

### 4.2 链路A —— 白条查账还款AI付

**用户侧流程：**

```
用户 → 京小贝Agent：query「查看我的白条账单」
京小贝Agent → 白条Agent：转发意图
白条Agent → 用户：展示查账还款卡片
用户 → 完成查账分期
用户 → 白条Agent：确认还款
白条Agent → 完成收单和写账流程
白条Agent → 京小贝Agent：识别为还款意图，返回支付参数+场景参数
京小贝Agent → 支付Agent：识别支付意图，透传支付参数和场景参数
支付Agent → AI付：获取AI付信息
AI付 → H5服务端：调用获取AI付付款方式
H5服务端 → 支付前置：调用获取AI付场景策略
支付前置 → 策略服务：返回策略信息
策略服务 → 支付前置：返回策略信息
支付前置 → H5服务端：返回并组装策略信息（付款方式、引导模式）
H5服务端 → AI付：返回AI付优先付款方式、支付方式、开通协议、按钮文案
```

### 4.3 链路B —— 一句话白条还款

```
用户 → 京小贝Agent：一句话引导「我要还白条」
京小贝Agent → 消金Agent：转发
消金Agent → 支付Agent：识别支付意图
支付Agent → AI付：获取AI付信息
AI付 → H5服务端：获取AI付付款方式
H5服务端 → AI付：返回收银台卡片
AI付 → 支付Agent：返回支付结果
```

### 4.4 支付结果

**成功：** 支付成功回调→展示支付结果卡片

**失败兜底：**
- **无可用支付工具：** 「没有可用的支付方式，您可以前往收银台继续支付」→跳转至还款页引导重新还款url（传入还款url）
- **非金融、京东等已集成支付SDK环境：** →「去还款」→跳转至还款页引导重新还款url（白条传入）

## 5. 核心功能差异

### 5.1 与已有AI付功能对比

| 维度 | 已有AI付 | 本期京小贝接入AI付 |
|------|----------|-------------------|
| **场景** | 金络收银台（通用支付） | 京小贝场景（白条还款闭环） |
| **入口** | 收银台SDK | 小贝MA→消金Agent→支付Agent |
| **支付工具偏好** | 银行卡占比90%（小金库转入50%） | 银行卡、银行转账、钱包余额、小金库 |
| **支付工具范围** | 支持分期 | 本期**不支持分期、组合支付 |
| **免密** | 部分场景免密 | 本次**不免密** |
| **SOP流程** | 标准收银台流程 | 接入SOP流程，串联小贝MA |
| **轮询** | 支持 | **不轮询**扣款 |
| **核验方式** | 支付密码/自研人脸 | 支持短信签约降级到支付密码 |
| **语音TTS** | 支持 | **不支持**TTS播报 |
| **卡片交互** | 标准卡片 | 支持点击/query双模式，代答模式 |
| **10轮澄清** | 无 | **支持最多10轮**，超10轮后兜底 |

### 5.2 新增功能点

| 新增功能 | 说明 |
|---------|------|
| **新增支付工具卡片 | 无可用支付工具→推荐新绑卡 |
| 切换支付工具卡片 | 用户主动切换、支付失败后引导切换 |
| 支付结果卡片（支付后推荐） | 支付后问题推荐 |

### 5.3 新增支付工具的场景

| 场景 | 卡片节点 | 支持添加模式 |
|------|---------|-------------|
| 无可用支付工具 | AI付收银台卡片 | 推荐 |
| 支付前/中/后用户主动新增 | 不限 | 推荐 / 用户触发 |
| 支付后推荐 | AI付支付结果卡片 | 推荐 |

## 7. 常见问题

### Q1：京小贝接入AI付需要什么前置条件？

A1 京小贝AI助手已经接入，用户通过小贝MA进入支付场景。需确保用户已开通京东支付账户，且当前场景配置为「不免密、不轮询扣款」。

### Q2 京小贝场景下接入AI付后，卡片交互是什么样的？

A2 AI付收银台卡片支持点击和query双模式：
- 点击卡片：在原卡片刷新
- 用户query（语音/文字）：唤起新卡片
- 历史卡片（非最新卡片）均折叠为过期可用状态
- 不支持TTS播报
- 用户每轮query都需要过模型识别意图后返回对应的卡片信息

### Q3 无可用支付工具的兜底逻辑是什么？

A3 分为两套：
- **金融环境：**「没有可用的支付方式，您可以前往收银台继续支付」→跳转至还款页引导重新还款url
- **非金融环境：**「去还款」→跳转至还款页引导重新还款url（白条传入）

### Q4 支付失败后怎么处理？

A4 支付失败后分为两种情况：
- **支付失败后推荐数据：** 由前置提供（本次**不包含**）
- **支付结果卡片和支付后推荐：** 本期**不解耦**

### Q5 京小贝场景支持哪些支付工具？

A5 支持：银行卡、银行转账、钱包余额、小金库（开通&转入小金库）。**不支持**：分期、数字人民币、数字人民币授权、组合支付。

### Q6 支付成功后有什么推荐？

A6 支付成功后展示**支付结果卡片**，并在支付结果卡片上做**支付后推荐**（新增支付工具推荐）。
