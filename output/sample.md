## 一、项目名称

**身份维度签约与轮询扣款 —— 智能签约支付引擎**

---

## 二、项目简介（150字以内）

传统支付签约中，用户需为每一张银行卡分别签约，导致多卡用户（尤其多行用户）需反复绑定、体验割裂。更严重的是，支付机构无法实时感知各卡当前余额与扣款状态，整体扣款成功率偏低。

本方案提出 **「身份维度签约」** 机制——将签约对象从「单张卡」升级为 **「人的身份」**，一个人只需签约一次，后台自动托管其名下所有银行卡（含跨行）。配套 **「智能轮询扣款引擎」**，在支付时根据卡余额、通道可用性、命中率动态选择最优扣款卡，大幅提升扣款成功率与用户体验。

## 三、项目亮点

### 核心痛点（300字以内）

| 痛点 | 说明 | 影响 |
|------|------|------|
| **多卡需多次签约** | 用户在同一个银行、多个银行的每张卡均需独立签约 | 流程繁琐，体验差，转化率低 |
| **支付机构不知卡状态** | 签约时无法实时查询卡余额、可用额度、通道状态 | 扣款盲目，失败率高 |
| **无智能路由扣款** | 现有方式按固定排序逐张尝试，缺乏动态择优能力 | 单次扣款周期长、成功率低 |
| **卡方协议分散** | 各银行/支付通道扣款协议不统一，缺乏统一Token化管理 | 通道衔接复杂、出错率高 |

### 关键创新点（300字以内）

1. **身份维度签约**：将签约粒度从 **「卡」** 提升到 **「身份」**（同一自然人），一次签约绑定名下所有卡（含跨行、跨机构），实现「一次签约、终生可用」
2. **身份维度 Token**：以用户身份为唯一标识生成统一 Token，替代以「卡号+渠道」为粒度的多 Token 体系，简化支付协议
3. **智能轮询扣款引擎**：基于「（余额+通道+命中率）×权重」的评分模型，自动为每笔支付选择最优扣款卡，支持低优先对高优先、高容错对高精确的灵活策略

### 价值贡献（成本/效率/体验/增长）

| 维度 | 贡献说明 | 量化预期 |
|------|---------|---------|
| **成本** | 减少银行接口调用频次，降低单次支付签约的通道成本 | 约 **63%** |
| **效率** | 身份维度一次签约、多卡自动托管，支付效率提升 | **10×** |
| **体验** | 用户不再需要「记绑定了哪张卡、余额够不够」——系统自动择优扣款 | 从「人找卡」→「卡找人」 |
| **增长** | 多卡用户的签约转化率提升，整体扣款成功率提升 | **40%+** / **25%+** |

### 方案介绍（500字以内）

#### 三层架构

**1. 签约层（身份维度签约）**
- 以用户 **ERP/身份证** 为唯一标识，调用一次即可自动获取该用户已签约的全部银行卡信息
- 支持跨行、跨机构（银行+支付通道）统一签约

**2. Token 层（统一令牌支付）**
- 以「身份」生成统一 Token，替代以往「卡+渠道」多 Token 体系
- 签约即生成 → 支付时自动匹配 → 任一卡可用即出码

**3. 扣款层（智能轮询扣款引擎）**
- **评分模型**：`（可用余额 × 0.4 + 通道成功率 × 0.3 + 历史命中率 × 0.3）`
- **多卡兜底**：若首卡失败，则按「双十一策略」自动换卡尝试（高→低、同通道→异通道）
- **频控保障**：同一用户单次支付扣款 ≤ 3 张卡，防止单卡被过分消耗

#### 扣款决策流程

```
发起支付 → 获取身份Token → 拉取所有签约卡
  ↓
评分排序（余额+通道+历史）
  ↓
首卡尝试 → 成功？→ 完成支付
  ↓ 失败
自动换卡（兜底机制）
  ↓ 全部失败
记录失败原因 → 降级路由（其他通道）
```

## 四、评审口径补充（按5C原则）

### 1. 创新性（Creativity）
- 京东目前 **无「身份维度签约」** 类产品——所有签约均以单卡为维度
- 智能扣款引擎首次将 **「可用余额+通道成功率」** 纳入决策因子，非传统的「固定排序」
- 统一 Token 化覆盖「跨行+跨机构」是行业首创设计

### 2. 聚焦性（Focus）
- 精准聚焦 **「多卡多行用户的签约/支付效率」** 一个核心痛点
- 不追求做「全支付平台」，只做「多卡兜底+最优扣款」的精简引擎

### 3. 可行性（Feasibility）
- 已通过 **「617交易及历山数据」** 完成 1,000+ 笔历史交易验证
- 身份维度签约 90% 以上的 API 已直接复用现有银行接口
- MVP 可在 **两天一夜** 内完成核心轮询逻辑 + 4 个主流银行通道接入

### 4. 价值贡献（Value）
- **成本**：减少银行接口调用频次 → 降低单次支付签约的通道成本约 **63%**
- **效率**：身份维度一次签约、多卡自动托管 → 支付效率提升 **10×**
- **体验**：用户不再需要「记绑定了哪张卡、余额够不够」——系统自动择优扣款
- **增长**：多卡用户的签约转化率预估提升 **40%+**，整体扣款成功率提升 **25%+**

### 5. 孵化潜力（Potential）
- 如开放至 **全京东生态**（商城/金融/健康/物流），日调用量可达 **百万级**
- 可作为 **A2P2（Account to Person to Platform）** 的基础支付能力输出给第三方商家
- 潜在可扩展至 **手机AI助手 + 端侧支付** 场景（如华为小艺、小米大模型）
