## 一、概述

### 1.1 需求背景及目标

**背景：** AI支付场景雷达每日自动更新一篇日报（Cron 18:00触发），当前页面仅展示最新一期，历史日报无法回溯查看。页面底部有「查看历史日报」链接但指向 `#`（未实现）。

**目标：** 让用户能够查看过往所有日报内容，支持按日期浏览，实现信息可追溯。

**历史版本：** https://xixixi-coder.github.io/ai-payment-radar/

### 1.2 期望上线时间

2026/6/18（当天完成，产品自驱需求）

### 1.3 预期收益ROI

| 指标 | 当前 | 预期 | 说明 |
|------|------|------|------|
| 页面回访率 | 低（每天只看最新） | +50% | 用户可回溯历史，增加粘性 |
| 内容利用率 | 仅当日可见 | 历史全量可查 | 历史沉淀的情报不再浪费 |
| 人工成本 | 0 | 0 | 纯前端实现，零额外维护 |

## 二、产品功能

### 2.1 涉及相关系统

| 涉及系统 | 上游依赖 | 需求内容 | 系统对接人 | 是否改造 |
|----------|----------|---------|-----------|---------|
| GitHub Pages | GitHub API | 读取历史commit中的index.html | 无（公开API） | 否 |
| ai-payment-radar repo | Cron Job | 每日commit保存日报 | 自己 | 是（归档策略） |

### 2.2 产品流程概述

```
用户进入雷达首页（展示最新日报）
  ↓
点击「查看历史日报」
  ↓
展示历史日报列表（按日期倒序）
  ↓
点击某一天 → 加载该日的日报内容
  ↓
可返回列表 / 返回最新
```

### 2.3 产品需求列表

| 产品模块 | 需求项 | 详细功能点描述 | 优先级 |
|----------|--------|--------------|--------|
| 归档 | 每日日报自动归档 | Cron更新时，将当日日报保存为 `archive/YYYY-MM-DD.html` | P0 |
| 列表页 | 历史日报列表 | 新页面 `history.html`，展示所有归档日报（日期+标题+摘要），按日期倒序 | P0 |
| 列表页 | 日期筛选 | 点击某日直接跳转到对应日报 | P0 |
| 入口 | 首页入口 | 将底部「查看历史日报」链接指向 history.html | P0 |
| 详情 | 历史日报查看 | 点击列表项 → 加载对应归档文件显示完整日报 | P0 |
| 导航 | 返回最新 | 历史页有「返回今日」按钮 | P1 |

无（公开页面，无需登录）

## 四、其他需求

### 4.1 技术方案

**归档策略（零成本）：**
- 每日 Cron 更新时，先将 `index.html` 复制到 `archive/YYYY-MM-DD.html`
- `history.html` 通过扫描 `archive/` 目录生成列表（纯前端，读取 GitHub API 获取文件列表）

**替代方案（更简单）：**
- 不用 GitHub API，直接在 Cron 任务中维护一个 `archive/index.json`（日期+标题列表）
- history.html 读取此 JSON 渲染列表

**选择：** 方案B更简单可靠，不依赖外部API。

### 4.2 性能需求

- 列表页加载 < 1s（JSON 文件小）
- 历史日报加载 < 2s（单文件 ~50KB）
